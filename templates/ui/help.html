Руководство пользователя

Платформа Платформа ВКР объединяет загрузку DOCX-шаблонов, справочники и настраиваемые бизнес-процессы (БП): специалист проектирует шаблон БП и правила полей, оператор запускает процесс и заполняет документы по шагам.

Роли

РольНазначение
АдминистраторПользователи, роли, общий доступ к документам и журналу.
СпециалистТипы документов, шаблоны DOCX, шаблоны БП, справочники, правила полей.
ОператорЗапуск БП, мастер шагов, реестр документов, наполнение справочников (строки данных); создание типов справочников и изменение структуры — у специалиста и администратора.

Администратор

  1. Создание учётных записей (страница регистрации или API).
  2. Назначение роли: ADMIN / SPECIALIST / OPERATOR (раздел «Пользователи» / «Роли»).
  3. Контроль действий — раздел «Журнал операций».

Специалист

Типы документов и шаблоны

  • Типы документов (/ui/document-types/) — список загруженных типов, поиск по названию, кнопка «Редактировать» для имени и описания типа.
  • «Поля» — подписи реквизитов, тип данных (текст, дата…), обязательность. В каталоге все поля хранятся как переменные; поведение «постоянное / автозаполнение» задаётся только в правилах БП.
  • «Загрузить шаблон» (/ui/templates/create/) — новый тип + DOCX; плейсхолдеры вида {% verbatim %}{{ ключ }}{% endverbatim %} извлекаются в реквизиты. Если в документе есть повторяющаяся таблица — отметьте чекбокс «В документе есть таблица» (см. раздел «Таблицы в DOCX»).
  • Предпросмотр типа — упрощённый (?simple=1) без классификации по БП; из карточки шага шаблона БП — с подсветкой постоянных и переменных полей по правилам этого шага (?bpt_id=…&pdt_id=…).

Шаблоны бизнес-процессов

  • Шаблоны БП (/ui/bpm/process-templates/) — создание шаблона, затем «Настроить БП».
  • В карточке БП: название и описание; этап можно добавить как новый тип + DOCX (с опцией «таблица») или выбрать существующий тип с уже загруженным DOCX.
  • Для каждого этапа: предпросмотр, загрузка/замена DOCX, «Правила» — откуда берётся значение каждого поля документа на этом шаге.

Правила полей (источники значений)

На странице правил доступны только три источника:

  • Ручной ввод — оператор вводит на шаге.
  • Справочник — значение из записи, выбранной при запуске (или из записи нужного справочника). Колонка выбирается из списка с русскими подписями, заданными при настройке колонок справочника.
  • Предыдущий документ — копирование из документа более раннего шага этого же БП.

После сохранения правил открывается снова карточка шаблона БП.

Справочники

  • Список (/ui/bpm/dictionaries/): специалист и администратор создают и удаляют справочники, настраивают колонки и главное поле; оператор открывает карточку и ведёт строки данных (без создания типа справочника и без изменения структуры).
  • В карточке справочника задаются название, колонки, главное поле (подпись при запуске БП; внутренний код строки формируется автоматически) и строки данных.
  • При необходимости в шаблоне БП указывается справочник объектов — запись при запуске порождает производственный объект для документов.

Прочее меню специалиста

  • Реквизиты — справочный просмотр по всем типам.
  • Разделы «Процессы», «Этапы», «Связи реквизитов», отдельные «Объекты» в основном сценарии не обязательны: сценарий выстраивается через шаблон БП и справочники.

Оператор

  • БП (/ui/bpm/run/) — выбор шаблона, «Запустить»; продолжение из таблицы «Мои экземпляры» (кнопка «Открыть»).
  • Запуск: если к шаблону привязаны справочники — нужно выбрать по записи в каждом (поиск). Если справочников нет — запуск без выбора; все поля заполняются на шагах. Отдельный выбор «производственного объекта» не показывается — объект строится из справочников или служебной записи.
  • Мастер шага: поля с подсказками (ручной ввод / автозаполнение). На последнем шаге «Завершить и сформировать DOCX» формируются все документы этапов и скачивается ZIP со всеми DOCX. Повторно скачать архив можно по ссылке на завершённом экземпляре (/ui/bpm/instance/<id>/download-all/).
  • Документы — реестр с указанием шаблона БП (если документ из процесса), предпросмотр и скачивание.
  • Справочники — наполнение строк в карточке справочника (без создания нового типа справочника).

Важные понятия

  • Постоянный / переменный в предпросмотре шага БП — не «тип в базе», а режим на этом шаге: автоподстановка по правилу (справочник, предыдущий документ) против ручного ввода.
  • Тип документа в каталоге — общий шаблон DOCX и набор полей; один тип может входить в несколько БП с разными правилами на разных шагах.

Подготовка DOCX

  • Формат DOCX.
  • Плейсхолдеры: {% verbatim %}{{ имя_поля }}{% endverbatim %} в тексте DOCX; допускаются и другие форматы подстановок, которые настроены в коде извлечения полей.
  • После загрузки ключи появляются как реквизиты; подписи и обязательность правятся на странице «Поля».

Таблицы в DOCX (повторяющиеся строки)

  • Обязательное оформление в Word: в таблице должны идти две подряд строки данных (не заголовок таблицы, а именно строки с плейсхолдерами), причём в обеих строках набор плейсхолдеров одинаковый (например, в каждой ячейке {% verbatim %}{{name}}, {{pod}}…{% endverbatim %} в той же структуре). Это сигнал для системы: «здесь шаблон строки таблицы». Ключи читаются и из обычной таблицы, и из вложенной (если Word поместил сетку внутрь ячейки).
  • При загрузке DOCX (отдельно — «Загрузить шаблон», или в БП — «новый тип + DOCX» / замена DOCX на шаге) включите чекбокс «В документе есть таблица». Тогда программа находит первую такую пару строк, сохраняет якорь таблицы и в каталог полей попадает по одному реквизиту на каждый уникальный ключ (ключи из двух строк не дублируются).
  • Тип документа помечается как шаблон с таблицей; на странице «Поля» у таких реквизитов отображается метка таблица.
  • Оператор при заполнении документа (мастер БП, «Создание документа», форма реквизитов) видит блок «Таблица»: может добавлять и удалять строки и вводить значения по колонкам; при генерации DOCX строки размножаются в документе по сохранённому шаблону строки.

Демо-данные

Команды из каталога backend:

  • python manage.py migrate — применить миграции.
  • python manage.py seed_demo — демо-справочники, шаблон БП, пользователи (см. вывод команды).
  • python manage.py reset_platform — очистка БД и повторная заливка демо.
